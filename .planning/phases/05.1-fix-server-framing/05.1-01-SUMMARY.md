# Phase 05.1 Plan 01: Integrate FramedTransport into Server Summary

## One-Liner

Integrated FramedTransport into NexoServer to enable proper length-prefix framing for all client-server communication, closing the critical integration gap identified in the v1.0 milestone audit.

## What Was Done

### Task 1: Update server to wrap TcpStream in FramedTransport on connection accept

Modified `spawn_connection_handler()` in `src/server/std_impl.rs` to:
- Wrap accepted TcpStream in `TokioTransport` and `FramedTransport`
- Pass `FramedTransport<TokioTransport>` to connection handlers
- Remove unused buffer allocation (FramedTransport handles buffering internally)

### Task 2: Replace raw stream.read() with framed.recv_message() for message receiving

Modified `handle_connection_with_dispatcher()` to:
- Use `framed.recv_message::<Casp001Document>()` instead of `stream.read(&mut buffer)`
- Update dispatcher call to pass decoded document
- Remove buffer parameter from connection handler methods
- Handle FramedTransport decoding errors properly

### Task 3: Add Dispatcher method for handling decoded message structs

Added `dispatch_document()` method to `src/server/dispatcher.rs`:
- Accepts decoded `Casp001Document` structs instead of raw bytes
- Eliminates double-encoding problem
- Keeps existing `dispatch(&[u8])` method for backward compatibility
- Added unit tests for the new method

### Task 4: Replace raw stream.write() with framed.send_message() for response sending

Modified connection handlers to:
- Use `framed.send_message(&response_document)` instead of `stream.write_all(&bytes)`
- Update dispatcher integration to use `Casp002Document` return type
- Ensure error responses are sent with proper framing
- Update echo fallback to use framing

### Task 5: Update heartbeat to use FramedTransport with proper CASP message

Updated `create_heartbeat_document()` to:
- Return `Casp001Document` instead of `Vec<u8>`
- Create proper CASP heartbeat with `MessageFunction="HRTB"`
- Use `framed.send_message()` for heartbeat transmission

### Task 6: Create E2E integration test with real client and real server

Created `tests/client_server_e2e_integration.rs` with:
- `test_e2e_client_server_payment_request` - Client sends payment, server responds
- `test_e2e_server_to_client_response_flow` - Multiple sequential requests
- `test_e2e_heartbeat_with_framed_transport` - Heartbeat framing verification
- `test_e2e_concurrent_clients` - 5 concurrent clients with real server
- `test_e2e_raw_framed_transport_communication` - Raw FramedTransport usage
- `test_e2e_length_prefix_framing_on_wire` - Wire-level framing verification
